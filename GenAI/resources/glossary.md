# Generative AI Glossary

## A

**Agent**
An AI system that can take actions autonomously to achieve goals. Can use tools, make decisions, and execute multi-step workflows.

**Attention Mechanism**
A technique allowing models to focus on relevant parts of input when generating output.

**API (Application Programming Interface)**
Interface for accessing AI models programmatically.

## B

**Beam Search**
Decoding strategy that explores multiple possible sequences simultaneously.

**BERT (Bidirectional Encoder Representations from Transformers)**
Model architecture that reads text bidirectionally for better understanding.

**Bias**
Systematic errors or prejudices in AI outputs, often reflecting biases in training data.

## C

**Chain-of-Thought (CoT)**
Prompting technique that encourages step-by-step reasoning.

**Checkpoint**
Saved state of a model during training.

**Context Window**
Maximum amount of text a model can process at once.

**Completion**
The AI-generated response to a prompt.

## D

**Diffusion Model**
Generative model type that creates data by gradually removing noise. Used for image generation.

**Discriminative Model**
Traditional AI model that classifies or predicts (opposite of generative).

## E

**Embeddings**
Numerical vector representations of text capturing semantic meaning.

**Epoch**
One complete pass through the training dataset.

## F

**Few-Shot Learning**
Providing a few examples in the prompt to guide model behavior.

**Fine-Tuning**
Training a pre-trained model on specific data to specialize it.

**Foundation Model**
Large pre-trained model serving as a base for various tasks.

## G

**GAN (Generative Adversarial Network)**
Model architecture with generator and discriminator competing.

**GPT (Generative Pre-trained Transformer)**
Popular LLM architecture (e.g., GPT-4).

**Grounding**
Connecting AI responses to verifiable sources or facts.

## H

**Hallucination**
When AI generates plausible but incorrect or fabricated information.

**Hyperparameters**
Settings that control model training and behavior (e.g., learning rate).

## I

**Inference**
Using a trained model to generate outputs (production use).

**Instruction Tuning**
Fine-tuning models to follow instructions better.

## L

**Latency**
Time delay between sending prompt and receiving response.

**LLM (Large Language Model)**
AI model trained on vast text data to understand and generate language.

**LoRA (Low-Rank Adaptation)**
Efficient fine-tuning method that modifies fewer parameters.

## M

**Masked Language Model**
Training approach where model predicts hidden words in sentences.

**Multimodal**
Models that process multiple content types (text, images, audio).

## N

**Neural Network**
Computing system inspired by biological brains, basis of modern AI.

**NLP (Natural Language Processing)**
Field of AI focused on language understanding and generation.

## O

**One-Shot Learning**
Providing exactly one example in the prompt.

**Overfitting**
When model learns training data too specifically, performing poorly on new data.

## P

**Parameters**
Learned weights in neural networks determining model behavior.

**Perplexity**
Metric measuring how well model predicts text (lower is better).

**Prompt**
Input text guiding AI to generate desired output.

**Prompt Engineering**
Craft of designing effective prompts for better AI outputs.

## R

**RAG (Retrieval-Augmented Generation)**
Enhancing LLMs by retrieving relevant information from external sources.

**Reinforcement Learning from Human Feedback (RLHF)**
Training method using human preferences to improve model behavior.

## S

**Semantic Search**
Search based on meaning rather than keyword matching.

**Stop Sequence**
Tokens signaling the model to stop generating.

**System Prompt**
Initial instructions setting model behavior and constraints.

## T

**Temperature**
Parameter controlling randomness in generation (0-2).

**Token**
Basic unit of text processed by models (~3-4 characters).

**Top-k Sampling**
Generation method considering only k most likely next tokens.

**Top-p Sampling (Nucleus Sampling)**
Generation method considering tokens up to cumulative probability p.

**Transfer Learning**
Using knowledge from one task to improve performance on another.

**Transformer**
Neural network architecture underlying most modern LLMs.

## V

**VAE (Variational Autoencoder)**
Generative model type using encoding and decoding.

**Vector Database**
Database optimized for storing and searching embeddings.

## Z

**Zero-Shot Learning**
Model performs task without any examples, just instructions.
